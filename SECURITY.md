# Security Policy

## Responsible Disclosure

Please report suspected vulnerabilities privately and do not disclose publicly until a fix is available.

## How to Report Vulnerabilities

Send a report to: security@project.invalid

Include:
- Affected versions and deployment details
- Reproduction steps / proof of concept
- Potential impact
- Suggested mitigations (if available)

We will acknowledge receipt within 3 business days and provide status updates as triage progresses.

## Security Policy

- We prioritize vulnerabilities that affect authentication, request routing, and remote code execution.
- Secrets should never be committed in issue reports.
- Security fixes may be released out-of-band from normal release cadence.
